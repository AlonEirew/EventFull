Overview

We aim to determine which events can be anchored in time, i.e., have a well-defined notion of time.
At this stage, you will see a paragraph with events marked in blue, indicating event mentions that require annotation.
For each event, you need to decide whether it can be anchored in time or not.

For example, given the sentence:

In Washington today, the Federal Aviation Administration released air traffic control tapes from the night the TWA Flight 800 went down.

Can the event released be anchored in time?
We say an event can be anchored on a real timeline if and only if:

  • Case 1: The event happened in the past or is happening now.
  • Case 2: The event is unconditionally planned to happen in the future.

Therefore, the correct answer is "Yes," and the event should be marked in green (using the mouse button).

When is an event mention not anchorable in time?

  • Case 1: The event is hypothetical or conditional (e.g., If I get the money, I will redecorate the house).
  • Case 2: The event is a negation (e.g., He did not attend the conference).
  • Case 3: The event is abstract or non-specific (e.g., It's amazing that a 975,000-pound Boeing 747-8 can fly).
  • Case 4: The event is static (e.g., any auxiliary verb).
  • Case 5: The event is recurrent (e.g., The team meet once a week for briefing).
  • Case 6: The event is an intention, wish, or opinion (e.g., Wouldn't it be wonderful if they start dating?).

Please note: To complete this step, all events must be marked as either anchorable or not.
Once this is done, the button will be enabled.
