Coreference Step - Tool Overview

You will see a paragraph with event mentions sharing an equal time (as selected in the previous temporal step). One event will be marked in green, and all others will be marked in red. You need to decide if these red event mentions refer to the same event in the world as the green one (as further explained below in "Task Overview").

After each selection, you can navigate to the next set of events requiring annotation using the button, or navigate sequentially through the sets of events using the and buttons. Once all event mention sets have been annotated, a notification indicating that the annotation process is complete will be displayed.

Note that at this step, manually selecting event mentions (nodes) from the graph visualization is not supported. However, you can move the nodes in the graph as you see fit.

In some cases, a change you make may implicitly impact other relations. In such cases, an Implicit Relation Update message will be displayed, and you will be required to review the changes before proceeding.

Task Overview

Below are examples of how to perform the coreference annotation task.

Example 1:

A traveler is kidnapped, and the police officers said he was taken two days ago.

[QUESTION] Which of the highlighted events refers to the same kidnapped event?

Select (if apply):

EXPLANATION: In this example, we have only a single event mention candidate that shares the same starting time as the event mention in focus. We need to determine if the event mentions kidnapped and taken describe the same real-world event. To make this decision, we must consider the arguments associated with both event mentions: the time, location, and participants. We observe that both mentions share the same participants (the kidnapped individual, "traveler/he") and the same time frame ("two days ago"), confirming that they indeed represent the same event.

IMPORTANT RULE

An event is usually associated with four components:

  • Action component: Refers to what happens or holds.
  • Time slot: Anchors the action in time.
  • Location component: Links the action to a place or location.
  • Participant component: Indicates the "who" or "what" is involved in the action.

Two events can be considered the same if and only if we can infer that they share the same action, time, location, and participant(s), as much as is indicated in the text and through the use of common sense.

Example 2:

Achilles was running in the left lane, and the turtle was running in the right lane.
